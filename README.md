## 🤖 AI-Native Engineering

> AI를 코딩 보조가 아니라 **엔지니어링 방식 자체**로 씁니다. 도메인 판단은 내가, 반복은 AI가 — 그 경계를 설계하고 팀에 배포했습니다.

```text
자동화 → 시간 확보 → 고민·판단 → 기준화 → AI 하네스 → 더 정교한 자동화 → 반복
```

- 🧩 **자체 워크플로우 인프라** — Claude Code 스킬 체계(`/start-day` · `/end-day` · `auto-doc` 등)로 문서화·트래킹·온보딩을 한 줄 명령으로 자동화
- 🔀 **멀티 에이전트 오케스트레이션** — cmux 기반 병렬 워크트리로 메인·서브 태스크를 동시 진행, 메인 세션은 설계·검토·지휘에 집중
- 📚 **지식 자산화 루프** — Obsidian 지식 저장소 + 웹 뷰어로 도메인 지식을 축적, 자동화가 지식을 쌓고 그 위에서 더 단단한 구현이 나오는 사이클 구축
- 🚀 **팀 확산** — 개인 워크플로우를 일반화해 팀에 배포, "한 번 고생한 것이 팀의 자산이 되는 구조"로 전환

<sub>"어떤 모델을 고르느냐보다, 그 모델이 올바르게 작동할 환경을 어떻게 설계하느냐가 더 중요한 질문이 되고 있습니다."</sub>
